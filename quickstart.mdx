## 1. Create your account

Sign in at [app.gethydra.ai](https://app.gethydra.ai) using your GitHub account.

## 2. Install the GitHub App

After signing in, install the Hydra GitHub App on the organization and repositories you want Hydra to scan. Hydra needs read access to your code and write access to open pull requests and post check results.

## 3. Add a repository

Go to **Repositories** and click **Add repository**. Select the repo and the branch you want Hydra to monitor.

Discovery runs automatically as soon as you add the repo.

## 4. Discovery runs

Hydra reads your entire codebase and builds a living knowledge base in `.hydra/`:

- `architecture.md` — how the codebase is structured
- `conventions.md` — naming patterns, test patterns, API patterns
- `howto/` — project-specific guides generated from your code

This runs once and takes 10–20 minutes depending on codebase size. You will see progress in the dashboard.

Hydra also opens a pull request to inject a block into your repo's `CLAUDE.md`. Merge it and every developer using Claude Code in that repo gets Hydra's architecture context in every session automatically.

<Info>
All AI inference runs on AWS Bedrock, managed by Hydra. No API key setup required.
</Info>

## 5. Run your first audit

Once Discovery completes, click **Run Audit**. The audit runs six parallel domain agent groups against your codebase and returns a prioritized list of findings. Each finding includes severity, effort estimate, risk rating, and a change plan.

Most repos complete in 10–20 minutes. Findings appear in the **Findings** tab sorted by priority.

## 6. Fix your first finding

Select a finding and click **Fix**. Hydra will:

1. Run the 11-gate quality gauntlet in an isolated Git worktree
2. Open a pull request with the fix
3. Close the Linear ticket on merge

<Check>
Your first autonomous fix is running. No further action required until you review the PR.
</Check>
