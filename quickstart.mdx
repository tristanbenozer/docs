# Install

Getting Hydra running against your codebase takes about ten minutes. There's no CLI, no agent to deploy, no infrastructure on your side — Hydra is a hosted product. You sign in, tell it which GitHub repos it can read, and you're done.

## 1. Sign in

Open the app and sign in with your work account.

- **Hosted:** Hydra is not currently deployed. The hosted apex (`app.<domain>` per env) lands with the [`infra/terraform/`](../technical/infrastructure.md) substrate.
- **Local dev:** `http://localhost:5173` if you're running the frontend yourself

You'll land on the dashboard. If your team already has a workspace, ask an admin to invite you. If you're the first one in, the workspace is created for you.

## 2. Install the GitHub App

Hydra reads code, opens pull requests, and posts review comments via a GitHub App that you authorize on your org or specific repos.

1. Go to **Settings → Integrations → GitHub**.
2. Click **Install Hydra GitHub App**.
3. On GitHub, pick the org and the repos you want Hydra to access. You can scope it to a handful to start and add more later.
4. Approve the install. You'll be redirected back to Hydra.

Hydra never stores GitHub OAuth tokens — every action against your repos uses a short-lived installation token issued by GitHub at request time.

## 3. (Optional) Bring your own Anthropic key

Hydra runs on Claude. By default, every audit, fix, and review runs on Hydra-managed inference (we host Claude on AWS Bedrock and bill you on the Hydra invoice) — there's nothing to configure to start running jobs.

If you'd rather pay Anthropic directly and see your usage on the Anthropic console, you can bring your own key:

1. Generate a key at [console.anthropic.com](https://console.anthropic.com).
2. In Hydra, go to **Settings → API keys**.
3. Paste the key and save. It's encrypted at rest.

When a key is configured, we route your audits through Anthropic's API using your key — your billing, your rate limit. Remove the key and we route through Hydra-managed Bedrock again. Either way, Hydra never proxies one tenant's traffic through another tenant's key.

## 4. Add your first repo

1. Click **+ Add repo** on the dashboard.
2. Pick a repo from the list (only ones the GitHub App can see show up).
3. Optionally pick a non-default branch — repos are tracked as `name@branch`, so you can audit `main` and `develop` independently.
4. Save.

Hydra immediately runs **discovery** on the repo — a one-time pass that learns the architecture, conventions, and stack. It takes 5–10 minutes and costs about $5–10 in Anthropic usage. Discovery is what makes every later audit and fix actually understand your codebase, so don't skip it.

## 5. (Optional) Link related repos

If two of your repos relate — a service and its tests, a producer and its consumer, a provider and its schemas — tell Hydra. Open the repo in the app, go to **Related repos**, and add each link with its relationship (`tests`, `shared-schemas`, `consumer`, `provider`, `sibling`).

Every audit, fix, and review on the linked repo will pick up the related repo's discovery context automatically. This is the single highest-leverage configuration step in the product, and it's free.
