Task accepts any freeform directive and executes it against your codebase. Unlike Fix (which resolves a detected finding) and Audit (which analyzes the codebase), Task starts from what you want done.

**Cost:** ~$1-10 on your Anthropic key. **Time:** 5-15 minutes depending on scope.

## How Task works

Hydra first runs a refinement step on your directive -- a separate model call that expands the raw instruction into a structured investigation and change plan. This step costs a small amount separately from the main execution and ensures the agent has a clear, scoped plan before touching any code.

The refined directive is then shaped into a synthetic finding and run through the same 11-step quality gauntlet as Fix. Task and Fix have identical quality controls.

## Example directives

```
Add rate limiting to all API endpoints in /src/services
```

```
Update all deprecated lodash calls to native equivalents across the codebase
```

```
Add JSDoc comments to all exported functions in the utils directory
```

```
Migrate the user service from callback-style error handling to async/await
```

```
Add input validation to the registration endpoint using Zod
```

## Scoping

Task performs best when scoped to a specific directory, service, or concern. Broad directives ("improve code quality") will be refined into something more specific during the refinement step, but you will get better results by being precise upfront.

<Tip>
If your directive would touch more than 20 files, break it into multiple Tasks. The diff size guard (20 files / 2,000 lines) will reject a fix that is too large, and you will spend time on a run that cannot complete.
</Tip>

## What Task produces

- A pull request with the changes and a description of what was done
- A testing plan in the PR description
- A `.hydra/fixes/{ref}/` directory committed into the branch with the original directive and change plan
- A Linear ticket created and closed on merge (if Linear is connected)

## Difference from Fix

| | Fix | Task |
|---|---|---|
| Input | A detected finding from an audit | A plain-English directive from you |
| Pre-flight | Validates the finding still exists | Runs directive refinement |
| Branch naming | `hydra/fix-{ref}` | `hydra/task-{ref}` |
| Quality gauntlet | Identical | Identical |
