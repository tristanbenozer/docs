title: "PR Review"
description: "Automated pull request review on every push. Two modes: fast deterministic or full LLM analysis with must-fix and consider tiers."
---

PR Review runs automatically when a pull request is opened or updated. It posts a single maintained comment that updates with every commit push.

## Two modes

### Scanner mode

**Cost:** ~$1-2 per PR. Deterministic pattern matching on the diff. Posts inline comments on specific lines. Fast, no LLM cost for the analysis itself.

Use scanner mode for high-volume repos where speed matters and you want to catch known anti-patterns without the latency of a full LLM analysis.

### LLM mode

**Cost:** ~$1-2.50 per PR. Full analysis using a Sonnet agent briefed with domain-specific checklists selected based on what the diff actually touches. Returns up to 7 findings per PR, tiered by severity.

**Must-fix findings** can be configured to block merge via the GitHub check system. **Consider findings** are advisory.

The review comment updates automatically as the developer pushes new commits. Resolved findings appear as strikethrough in the maintained comment.

## Domain-aware analysis

In LLM mode, the review agent does not run a generic checklist. It receives tool-specific content selected based on what files changed. A PR touching the authentication layer gets different review criteria than a PR touching the UI layer.

## Wide blast radius detection

Hydra automatically detects changes to high-risk paths: `middleware/`, `framework/`, `core/security/`, and `auth/`. These changes are flagged in the review comment as having wide blast radius -- ripple effects that scanner mode may miss -- and LLM mode is recommended.

## GitHub check integration

PR Review can post a GitHub check result (green check or red X) on every PR. When must-fix findings exist and merge blocking is enabled, the check will fail until the findings are resolved or dismissed.

Configure this in **Repository Settings > PR Review** in the Hydra dashboard.

## Manual review

Any open pull request can be reviewed on demand from the Hydra dashboard without waiting for a push event. Useful for reviewing PRs that were opened before Hydra was connected to the repository.

## Configuring PR Review

In **Repository Settings > PR Review**:

- Enable or disable automated review
- Choose default mode (Scanner or LLM)
- Enable merge blocking on must-fix findings
- Configure which branches trigger review
