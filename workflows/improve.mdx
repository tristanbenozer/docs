Improve runs continuous improvement scans across your codebase and generates suggestions for things that work but make the codebase slower to operate in. Each suggestion is dispatched into the same 11-gate fix pipeline as a regular Fix.

**Cost:** ~$7–11 in compute credits. **Time:** ~11 minutes.

## What Improve targets

Improve focuses on code quality — not bugs, not security issues. Dead code removal, naming conventions, structural refactors, documentation gaps, dependency cleanup, missing type hints, deep nesting, and extractable constants.

Improve findings use `I-` prefixes to distinguish them from audit findings (e.g., `I-001`, `I-002`). Each suggestion is dispatched directly into the fix pipeline — there is no separate queue or review stage before execution.

## What Improve produces

- A prioritized list of code quality improvements, each scoped and estimated
- For each selected improvement: a pull request with the change, a change plan, a testing plan, and a post-improvement audit to verify no regressions
- Linear ticket created and closed on merge

## When to use Improve

Use Improve when you want to address technical debt systematically. Improve operates on code quality — things that work but slow the team down. Fix operates on bugs and security issues. They do not overlap.
