Govern runs two things automatically: PR Review on every pull request, and scanner pattern generation after every resolved finding. No manual trigger required for either.

## PR Review

**Cost:** ~$0.50–2.50 per PR in compute credits. **Time:** Seconds (scanner mode) to minutes (LLM mode).

Govern runs PR Review automatically when a pull request is opened or updated. It posts a single maintained comment that updates with every commit push. Must-fix findings can block merge via the GitHub check system.

### Two modes

**Scanner mode** — Deterministic pattern matching on the diff. Posts inline comments on specific lines. Fast. Use this for high-volume repos where speed matters.

**LLM mode** — Full analysis using a Sonnet agent briefed with domain-specific checklists selected based on what the diff actually touches. Returns up to 7 findings per PR, tiered by severity.

### Domain-aware analysis

In LLM mode, the review agent does not run a generic checklist. It receives domain-specific content selected based on what files changed. A PR touching the authentication layer gets different review criteria than a PR touching the data layer.

### Wide blast radius detection

Hydra automatically flags changes to high-risk paths — middleware, core security, auth — as having wide blast radius and recommends LLM mode for review.

## Scanner pattern generation

Every finding Hydra resolves teaches the scanner something new. When a fix closes a finding, Govern analyzes the pattern and generates a reusable YAML scanner rule that catches the same class of issue instantly on future audits — at zero LLM cost.

New rules are added to your team's scanner pool and activate on the next audit. Rules can optionally be contributed to the global pattern library, making them available to all Hydra tenants. Global contributions send only mechanical fields: the regex, severity, file scope, and category. No code or finding text crosses the tenant boundary.

## CLAUDE.md injection

Govern also manages your repo's `.hydra/` knowledge base. When Discovery updates or conventions change, Govern opens a PR that updates your `CLAUDE.md` block with the latest architecture context. Merge it and every developer using Claude Code gets the updated context automatically.

## Configuring Govern

In **Repository Settings > Govern**:

- Enable or disable automated PR Review
- Choose default mode (Scanner or LLM)
- Enable merge blocking on must-fix findings
- Configure which branches trigger review
- Set severity thresholds per repo
