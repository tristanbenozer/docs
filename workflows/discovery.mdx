title: "Discovery"
description: "Hydra reads your entire codebase once, builds an application profile, and generates the documentation that every subsequent workflow runs on."
---

Discovery is the first action Hydra takes on any repository. It runs automatically when you add a repo and runs once. Everything else -- audits, fixes, PR reviews -- uses the context Discovery produced.

**Cost:** ~$5-10 on your Anthropic key. **Time:** 10-20 minutes depending on codebase size.

## What Discovery produces

Discovery generates two outputs: an internal context used by Hydra's analysis and fix agents, and a published output deployed to your repository via pull request.

### Internal context

- Application profile: what type of application this is (customer-facing, internal tool, external API, etc.), its audience, data sensitivity level, and compliance requirements
- Architecture overview: how the codebase is structured, which components own which responsibilities
- Conventions: how features are built in this codebase, naming patterns, test patterns, API patterns
- How-to guides: project-specific guides Hydra generates from the code (e.g. "how to add a new API endpoint in this repo," "how to write tests for this service")

### Published output: the `.hydra/` directory

Discovery opens a pull request to your repository adding a `.hydra/` directory containing:

```
.hydra/
├── hydra.md          # Table of contents and entry point for AI agents
├── architecture.md   # How the codebase is structured
├── conventions.md    # Patterns found in this codebase
└── guides/           # Project-specific how-to guides
```

Merge the PR to activate the full Discovery output.

## Application profiling

During Discovery, Hydra determines the profile of the application and uses it to weight all subsequent analysis. The same finding gets a different priority depending on what kind of application it was found in:

| Application type | What gets weighted higher |
|---|---|
| Customer-facing frontend | Accessibility, PII handling, security |
| External API | Input validation, authentication, rate limiting |
| Internal tool | Performance, observability, maintainability |
| Background service | Reliability, error handling, resource management |

This weighting applies to every audit and PR review on the repository. It is not configurable -- Hydra determines the profile from the code.

## CLAUDE.md injection

Discovery also patches the repository's `CLAUDE.md` file (creating it if it does not exist) with a reference to `hydra.md`. After the Discovery PR merges, every developer on the team using Claude Code will automatically load Hydra's codebase context when they work in the repository -- without installing anything or changing how they work.

<Info>
CLAUDE.md injection is passive distribution. One developer installs Hydra. The entire team benefits from the generated documentation in their AI coding sessions automatically.
</Info>

## Re-running Discovery

Discovery does not run automatically after the initial run. To regenerate the `.hydra/` directory after significant codebase changes, trigger a manual re-run from the repository settings in the Hydra dashboard.
