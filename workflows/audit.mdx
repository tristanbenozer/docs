The Audit workflow runs a full analysis of your codebase and returns a prioritized list of findings. Discovery must complete before Audit can run.

**Cost:** ~$3–15 in compute credits depending on codebase size. **Time:** 10–20 minutes.

## How the audit works

### Pre-flight: repo-specific protocol generation

Before any analysis runs, a model call generates an analysis protocol tailored to this specific repository. Every agent is briefed on the repo's profile, constraints, and historical patterns before starting. Agents do not run generic checklists — they run checklists customized to this codebase for this run.

### Step 1: Deterministic scanner

Hydra runs a library of 39 deterministic scanner patterns against the codebase — Semgrep-backed regex patterns over specific file types evaluated in parallel. Fast, zero LLM cost. Scanner findings are capped at `medium` severity until LLM-verified. Pattern matches alone cannot raise a `critical` or `high` finding.

### Step 2: Six parallel domain agent groups

Six agent groups run simultaneously, each covering a domain. Every finding produced includes severity, effort estimate, risk rating, confidence level, and a change plan.

| Group | Analysis areas |
|---|---|
| **Security** | Auth, access control, hardening, privacy, compliance |
| **Correctness** | Dead code, naming, structural refactors, AI usage patterns, test coverage |
| **Maintainability** | Coupling, complexity, documentation gaps, dependency hygiene |
| **Performance** | Concurrency, resilience, resource management, cost footprint |
| **Architecture** | API contracts, schema design, database patterns, input validation, multi-tenancy |
| **Operations** | Dependencies, config management, observability, logging, feature flags |

All six groups run in parallel. Combined output is a deduplicated, prioritized finding pool.

## Findings

Each finding contains:

| Field | Description |
|---|---|
| `severity` | `critical`, `high`, `medium`, or `low` |
| `effort` | `trivial`, `small`, `medium`, or `large` — how hard the fix is |
| `risk` | `low`, `medium`, or `high` — risk of attempting the fix |
| `confidence` | `high`, `medium`, or `low` — agent's self-assessed evidence quality |
| `change_plan` | Agent-authored description of the approach, files to modify, and test strategy |
| `source` | `scanner` (deterministic) or `agent` (LLM) |

See [Finding Fields](/reference/finding-fields) for the complete schema.

## Freeform Notes

If you know something about the codebase that Discovery would not surface, add it as a Freeform Note on the repository. Notes inject directly into every LLM prompt Hydra runs on that repo — immediately active, no re-run required.

Examples:
- "We are mid-migration from PostgreSQL to CockroachDB. Do not suggest PostgreSQL-specific patterns."
- "The auth module is being replaced in Q3. Flag issues but do not fix."
- "This repo targets WCAG 2.2 AA. Weight accessibility findings accordingly."

## Running an audit

Audits are triggered manually from the repository dashboard. After an audit completes, findings appear in the **Findings** tab sorted by a computed priority score.
